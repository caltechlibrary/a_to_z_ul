Write an A to Z web component call a-to-z-ul who’s innerHTML contains a UL list and display the same list with links by alphabetical section.

The web component will be in a file called “a_to_z_ul.js”. Three example HTML files will demonstate the web component. The first one example HTML with a short list. The second example HTML file with a long list demonostrating how the link to menu will work. The third HTML file that include two separate a-to-z-ul lists with to demonstrate that the linking between an index heading and the index menu are independent between the lists.

The A to Z index should use a “menu” element. Each menu li should contain a link to that an anchor element in the UL list displayed. The element anchor title should link back to the menu.

If a list is long an optional link at the bottom of the web component should take you back to the web component’s menu element. What is considered long is configurable. The default item count for when the back to menu link shows up is 50. The attribute in the a-to-z-ul element is called long-count.

The web component should use HTML templates for structure and critical CSS. The web compent should inherit styling from the page it is embedded for the ul list, menu and menu’s li item.

The UL items should be taken form the a-to-z-ul innerHTML. The menu li should not have bullets and should be displayed inline.

The innerHTML of the a-to-z-ul should be a UL list used to construct the UL list presented by a-to-z-ul.

The a-to-z-ul web component may exist multiple times on a page.

The a-to-z-ul should be implemented as an ES6 module.
